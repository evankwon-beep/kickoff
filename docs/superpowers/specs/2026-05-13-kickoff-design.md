# 킥오프 (Kickoff) — 디자인 명세

작성일: 2026-05-13
대상 사용자: 한국 축구 팬 (운영자 본인 + 친구 등 일반 공개)

## 1. 개요

한국 축구 팬을 위한 **유럽 4대 리그 + 메이저 컵 대회 종합 대시보드 사이트**.
순위, 일정/결과, 하이라이트, 한국 선수 활약을 한 화면에서 보고, 경기를 클릭하면 라인업·통계·하이라이트 임베드까지 들어갈 수 있다.

비주얼은 **다크 모드 + 프리미엄 모던** 톤. 모바일/PC 반응형.

## 2. 핵심 결정 요약

| 항목 | 결정 |
|---|---|
| 사이트 성격 | 공개 사이트 (인터넷 호스팅, 친구 공유) |
| 메인 화면 | 종합 대시보드 (순위 + 일정 + 하이라이트 한 페이지) |
| 데이터 갱신 | 접속 시 갱신 + 1시간 캐시 (라이브 스코어 X) |
| 대회 범위 | 4대 리그 + UEFA(챔스/유로파/컨퍼) + FA컵 + 메이저 국대(월드컵/유로/코파/아시안컵). 단 무료 API 제약으로 1차 출시 시 **유로파/컨퍼런스, 각국 컵(코파 델 레이/DFB 포칼/코파 이탈리아)은 제한적/제외**. 데이터 소스 추상화로 추후 유료 전환 시 모든 대회 활성화 |
| 디바이스 | 모바일/PC 반응형 |
| 시각 톤 | 다크 모드 + 프리미엄 모던 (블랙 + 화이트 + 라임/네온 그린 강조) |
| 한국 특화 | 한국 선수 강조(🇰🇷 배지) + 한국 시간대(KST) 자동 변환 |
| 즐겨찾기/회원 | 없음 (단순화) |
| 비용 전략 | 무료로 시작, 데이터 소스 추상화로 추후 유료 전환 가능 |
| 선수 평점 | 자체 미제공. 통계로 대체 + FotMob 외부 링크 보조 |

## 3. 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | **Next.js (App Router)** | React 기반, Vercel과 1클릭 배포, ISR로 캐시 단순 |
| 호스팅 | **Vercel 무료 티어** | 무료 한도 내에서 충분, 자동 배포. 잠정 주소: `kickoff.vercel.app` (커스텀 도메인은 추후 결정) |
| 스타일 | **Tailwind CSS** | 다크 모드 기본 지원, 빠른 작업 |
| 폰트 | **Pretendard** (KR) + **Inter** (EN) | 한글/영문 가독성 |
| 데이터(축구) | **Football-Data.org 무료** + **API-Football 무료(보조)** | 무료 한도 내에서 풀세트 근사 |
| 데이터(영상) | **YouTube Data API v3** | 쿠팡플레이 채널 영상 가져오기 |
| 캐시 | **Next.js ISR (revalidate = 3600초)** | 1시간마다 자동 재생성, 사용자 부하 없음 |

### 데이터 소스 분담

- **Football-Data.org** (기본):
  - 순위, 일정/결과, 팀 기본 정보, 팀 엠블럼 URL, 선수단 명단
  - 4대 리그, 챔피언스리그, 월드컵, 유로, 코파 아메리카, FA컵
- **API-Football** (보조, 무료 100req/일):
  - 경기 상세의 라인업, 골/카드 타임라인, 점유율/슈팅 등 통계
  - 캐시 적극 활용 (한 번 가져온 끝난 경기는 영구 캐시)
- **YouTube Data API v3**:
  - 쿠팡플레이 채널의 최근 업로드 영상 목록
  - 보조 채널(SPOTV 등) 폴백
- **자체 정적 데이터**:
  - `data/korean-players.json` — 손흥민/이강인/김민재 등 한국 선수 명단 (운영자가 수동 관리)

### 유료 전환 가능성

모든 외부 API 호출은 `lib/dataSource/` 인터페이스 뒤에 가둔다. 추후 라이브 스코어, 평점, 빠진 컵대회가 필요해지면 이 인터페이스 구현체만 교체하면 된다 (예: API-Football 유료 플랜, SportMonks 등).

## 8. 컴포넌트 설계

### 데이터 레이어 (`lib/dataSource/`)

추상 인터페이스 — 추후 유료 API로 갈아끼울 때 이 인터페이스만 유지하면 됨.

```
interface DataSource {
  getStandings(leagueCode): Promise<Standings>
  getFixtures(opts): Promise<Fixture[]>
  getMatch(id): Promise<MatchDetail>
  getTeam(id): Promise<TeamDetail>

```

- `footballData.ts` — Football-Data.org 어댑터 (기본)

- `composite.ts` — 위 셋을 조합해서 위 인터페이스를 충족

## 9. 한국 특화 구현

- **시간대 변환**: 모든 경기 시간을 KST(`Asia/Seoul`)로 변환.
  - 절대 시간: `2026-05-14 04:00 (수)`
  - 상대 시간: `오늘 새벽 4시`, `내일 22:00`, `3일 후`
  - 유틸: `lib/time.ts` — `formatKST(date)`, `formatRelative(date)`
- **한국 선수 명단**: `data/korean-players.json`
  ```json
  [
    { "id": "son-heung-min", "name": "손흥민", "nameEn": "Son Heung-min",
      "teamId": "tottenham", "position": "FW" },
    ...
  ]
  ```
- **경기 행의 🇰🇷 배지**: 라인업이 있으면 라인업 기반, 없으면 "선수 소속팀이 그 경기에 출전" 추정
- **한국 선수 섹션**: 이번 주(최근 7일) 출전 경기 + 골/도움 자동 집계

## 10. 하이라이트 영상 매칭

1. YouTube API로 쿠팡플레이 채널(`@CoupangPlay` 등) 최근 50개 영상 목록 가져옴
2. 보조 채널(SPOTV NOW 등) 폴백
3. 영상 제목에서 키워드 추출 (팀명 한글/영문 모두 시도, 정규화)
4. 경기 데이터의 두 팀 이름과 양쪽 매칭되면 연결
5. 매칭 실패 시 "곧 업로드 예정" 표시
6. 매칭은 빌드 타임에 1회 + ISR 갱신 시 갱신

## 11. 데이터 흐름

```
[Football-Data.org] ──┐
[API-Football]       ──┼──> Next.js ISR 빌드/리밸리데이션
[YouTube API]        ──┘     (1시간마다)
                              │
                              v
                         캐시된 HTML
                              │
                              v
                          사용자 브라우저
```

- 페이지 첫 빌드 시 전부 페치 → 정적 HTML 생성
- 1시간 지난 페이지 요청 → 백그라운드 재빌드 (사용자는 캐시된 페이지 즉시 받음)
- API 호출은 서버 측에서만 발생 → API 키 브라우저 노출 X

## 12. 에러 처리

- **외부 API 전체 실패**: 마지막 캐시된 페이지를 그대로 보여줌. 상단에 작은 토스트로 "데이터 갱신 실패 — 마지막 업데이트: X시간 전"
- **부분 실패 (예: API-Football 다운)**: 라인업/통계 섹션만 "데이터 없음" 메시지, 나머지는 정상
- **하이라이트 매칭 실패**: 해당 경기에 "곧 업로드 예정"
- **시즌 휴식기**: 가장 가까운 다가오는 경기까지 카운트다운

## 13. 테스트

최소한 다음은 단위 테스트로 보장:

- 한국 시간대 변환 (`formatKST`, `formatRelative`)
- 유튜브 영상 ↔ 경기 매칭 로직 (제목 기반, 한글/영문 정규화)
- 데이터 소스 어댑터 응답 파싱 (mock fixtures로)
- 한국 선수 배지 부착 로직

E2E는 일단 패스 (무료 사이트 규모상 과한 투자).

## 14. 향후 확장 여지

- 라이브 스코어 (SSE/WebSocket + 유료 API)
- 자체 선수 평점 (유료 API 또는 자체 알고리즘)
- 즐겨찾기 팀 (LocalStorage 또는 Supabase 무료 DB)
- 알림 푸시 (Web Push)
- 다국어 (영어/일본어)
- PWA (모바일에 앱처럼 설치)

## 15. 비기능 요구사항

- **성능**: 첫 로딩(LCP) 2초 이내 (정적 HTML이라 충분히 가능)
- **접근성**: 키보드 네비게이션, alt 텍스트, 충분한 색 대비
- **SEO**: 메타 태그, 페이지별 title/description, sitemap
- **분석**: Vercel Analytics 무료 티어 또는 Plausible
